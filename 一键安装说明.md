## Claude Code + Sub-agent 多模型 Agent 工作流（进阶）

一键安装工具只负责 Python、uv、VSCode 等**基础开发环境**，不会直接写入 Claude Code 配置或任何 API Key。

本项目同时提供 **Sub-agent + CCR 开箱即用工作流**。完成一键安装并解压项目后，运行重新初始化：

```bat
重新初始化 V1.24.bat
```

初始化会把 `.claude_templates/agents` 中的 5 个基础模板（architect / implementer / reviewer / tester / docs-writer）同步到 `%USERPROFILE%\.claude\agents`。基础模板默认 `model: inherit`，不写入 API Key，不会覆盖已有自定义 agent。

如需为不同角色绑定具体模型（Claude 原生、DeepSeek、Kimi、Ark Coding Plan 等），在初始化成功后运行：

```bat
python src\sub_agent_ccr_model_config.py
```

### 关于火山方舟 Coding Plan（可选）

Ark Coding Plan 是独立的多模型编码 provider，**与普通豆包 API 不兼容**：

| 特性 | 普通豆包（doubao） | 火山方舟 Coding Plan（ark_coding_plan） |
|---|---|---|
| Base URL | `https://ark.cn-beijing.volces.com/api/v3` | `https://ark.cn-beijing.volces.com/api/coding/v3` |
| API Key | 普通方舟 Key | Coding Plan 专属 Key |
| 模型列表 | doubao-seed-* 等 | kimi-k2.5 / deepseek-v3.2 / doubao-seed-code-preview-latest 等 |

配置 Ark Coding Plan：
1. 在 CCR config 中新增 `ark_coding_plan` provider，base_url 指向 Coding Plan 入口，填入 Coding Plan API Key。
2. 在 `src\sub_agent_ccr_model_config.py` 中选择 CCR 模式，provider 选 `ark_coding_plan`。
3. 本仓库**不保存、不读取任何 API Key**，CCR 鉴权由用户自行管理。

`ark-code-latest` 通过控制台统一管理模型版本，切换后约 **3-5 分钟**生效，不推荐作为日常快速切换默认值。

参考模板：`.claude.env.example`（仅作说明，不会被代码自动读取）。
